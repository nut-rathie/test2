# สรุปวิดีโอ: การใช้ Git สำหรับคนที่ไม่ใช่โปรแกรมเมอร์

## ภาพรวม
วิดีโอนี้สอนวิธีการใช้ Git สำหรับบันทึกสถานะของโปรเจค (snapshot) เพื่อย้อนกลับได้เมื่อเกิดปัญหา เช่น AI แก้ไฟล์จนพัง สามารถใช้ฟรีในเครื่อง และ sync กับ GitHub ออนไลน์ได้

---

## ส่วนที่ 1: แนะนำเครื่องมือ

### Git คืออะไร
- เครื่องมือบันทึกสถาวะโปรเจค (save snapshot)
- ย้อนกลับได้เมื่อไฟล์พัง
- ใช้ฟรีในเครื่อง
- Sync กับ GitHub ออนไลน์ได้

### ใครควรใช้
- ไม่จำเป็นต้องเป็นโปรแกรมเมอร์
- ใช้กับไฟล์ได้ทุกประเภท (text, รูป, Excel, ฯลฯ)

---

## ส่วนที่ 2: เริ่มต้นใช้งาน

### การทำงานร่วมกับ AI
- ไม่ต้องจำคำสั่ง Git ที่ยุ่งยาก
- ให้ AI ช่วยทำแทนได้ เช่น "ช่วยทำให้โปรเจคนี้รองรับ Git หน่อย"

### ขั้นตอนเริ่มต้น
1. **Initialize** - AI จะสร้าง Git repository ให้
2. **สร้าง .gitignore** - บอกว่าไฟล์ไหนไม่ต้องบันทึก
   - AI จะแนะนำไฟล์ที่ไม่ควร track ให้
   - เช่น ไฟล์ temporary, dependencies ฯลฯ

---

## ส่วนที่ 3: การ Commit

### Commit คืออะไร
- การบันทึก snapshot ปัจจุบัน
- เหมือนการเซษจุดที่สามารถย้อนกลับมาได้

### ตัวอย่างการใช้งาน
1. สร้างไฟล์ใหม่ (เช่น test.txt, CL.md)
2. แก้ไขไฟล์ (เพิ่มข้อความ เปลี่ยน content)
3. **บอก AI ว่า "commit ด้วย"**
   - AI จะ check ว่ามีอะไรเปลี่ยนแปลงบ้าง
   - เพิ่มไฟล์ (u = untrack คือไฟล์ใหม่)
   - แก้ไขไฟล์ (m = modify)

---

## ส่วนที่ 4: การย้อนกลับ (Restore/Reset)

### กรณีไฟล์พัง
- ถ้าแก้ไฟล์ผิด หรือ AI แก้จนไฟล์พัง
- บอก AI: "ไฟล์พัง ช่วย reverse กลับไป commit ล่าสุด"
- AI จะ restore กลับมาเหมือนเดิม

### การดูประวัติ
- บอก AI: "commit ก่อนหน้ามีอันไหนบ้าง"
- AI จะ list ให้ดูว่ามี commit อะไรบ้าง
- เลือกย้อนกลับไปตอนที่ต้องการ

### 2 วิธีในการย้อนกลับ
1. **สร้าง commit ใหม่ที่ย้อนกลับ** - เก็บ history เดิมไว้
2. **Hard reset** - ลบ commit ล่าสุดทิ้งไปเลย

---

## ส่วนที่ 5: การ Sync กับ GitHub

### สร้าง Repository บน GitHub
1. ไปสมัครที่ GitHub ก่อน
2. สร้าง Repository ใหม่
   - ตั้งชื่อ (เช่น test project)
   - เลือก public (คนอื่นเห็นด้วย) หรือ private (เห็นคนเดียว)
   - กด create repository

### เชื่อมต่อกับ GitHub
1. **Copy URL** ของ repository
2. **บอก AI**: "อยาก sync กับ GitHub อันนี้ของผม"
3. **AI จะทำ**:
   - Add remote เข้าไป
   - Push ขึ้นไป (ดันจากเครื่องขึ้นออนไลน์)

### การติดตั้งครั้งแรก
- ต้อง setup User/password หรือ SSH key ก่อน
- AI จะแนะนำวิธี:
  - ใช้ Git CLI
  - สร้าง SSH Key
  - เพิ่ม Public key ใน GitHub settings

---

## ส่วนที่ 6: การจัดการไฟล์ด้วย .gitignore

### เพิ่มโฟลเดอร์ที่ไม่ต้องการ
- ถ้าไม่อยากเอาโฟลเดอร์ (เช่น image) ขึ้น GitHub
- บอก AI: "ไม่อยากเอาโฟลเดอร์ image เข้าไปด้วย มันใหญ่มาก"
- AI จะเพิ่มเข้าไปใน .gitignore ให้

### Untrack ไฟล์ที่ถูก track แล้ว
- ถ้าไฟล์ถูก push ไปแล้ว แต่อยากเอาออก
- บอก AI ให้ untrack
- ไฟล์จะยังอยู่ใน local แต่ไม่ถูก track อีกต่อไป
- ใน GitHub จะหายไป

---

## ส่วนที่ 7: เคล็ดลับการใช้งาน

### ให้ AI ช่วย
- ไม่ต้องจำคำสั่งทั้งหมด
- ถ้าสงสัยอะไร ถาม AI ให้อธิบาย
- สังเกตคำสั่งและคำศัพท์เอาไว้นิดหน่อย
- ขอให้ AI list คำศัพท์สำคัญๆ ให้เรียนรู้ได้

---

## คำศัพท์สำคัญ (Keywords)

| คำศัพท์ | ความหมาย |
|----------|------------|
| **Git** | เครื่องมือ version control ในเครื่อง |
| **GitHub** | บริการเก็บ repository ออนไลน์ |
| **Repository (Repo)** | โฟลเดอร์/โปรเจคที่ใช้ Git |
| **Commit** | การบันทึก snapshot |
| **Snapshot** | สถานะของไฟล์ตอนนั้น |
| **Initialize** | เริ่มต้นใช้งาน Git ในโปรเจค |
| **.gitignore** | ไฟล์บอกว่าอะไรไม่ต้องบันทึก |
| **Restore** | คืนค่าไฟล์กลับมา |
| **Reset** | ย้อนกลับไป commit ก่อนหน้า |
| **Remote** | การเชื่อมต่อกับ GitHub |
| **Push** | ดันข้อมูลขึ้น GitHub |
| **Track/Untrack** | การติดตาม/ไม่ติดตามไฟล์ |
| **Modify** | การแก้ไขไฟล์ |
| **Untrack** | ไฟล์ใหม่ที่ยังไม่เคยถูกบันทึก |

---

## สรุปท้ายวิดีโอ
Git เป็นเครื่องมือที่มีประโยชน์มาก ไม่จำเป็นต้องเป็นโปรแกรมเมอร์ก็ใช้ได้ ใช้กับไฟล์ได้ทุกประเภท และสามารถให้ AI ช่วยทำแทนได้ ทำให้การใช้งานง่ายขึ้นมาก
